export const quickstart = `# In any repo with an afk.Dockerfile:
afk init --provider aws                                    # point AFK at your cloud
afk golden build                                           # one-time: build the boot image
afk secrets put github-token <PAT>                         # so the Run can clone your repo
echo "GITHUB_TOKEN=secret:github-token" >> .afk.env
afk run "claude -p 'fix the failing test and open a PR'"   # launch an agent in your cloud
afk ls                                                     # see it running
afk logs <run-id>                                          # tail its output`

<Code code={quickstart} lang="sh" title="Your first Run" />

That Run boots an ephemeral instance in your own AWS account, does the work,
and terminates. The same commands work unchanged on GCP and Cloudflare.

## Why AFK

<CardGrid stagger>
  <Card title="One CLI to rule your clouds" icon="laptop">
    The same `afk run`, `afk attach`, `afk ls`, `afk kill` on AWS, GCP, and
    Cloudflare. Switch providers without changing a command.
  </Card>
  <Card title="Ephemeral by design" icon="rocket">
    Each Run boots, does its work, and self-terminates. No always-on
    infrastructure to maintain or pay for.
  </Card>
  <Card title="Your dev environment, ported" icon="puzzle">
    A Dockerfile and compose file describe the Run — so your entire dev and
    test environment (toolchain, Postgres, Redis, …) travels to the cloud and
    is available to the agent.
  </Card>
  <Card title="Stay in the loop" icon="approve-check">
    Tail an agent's logs live, attach a shell into its container to steer it,
    or kill it — from wherever you are.
  </Card>
</CardGrid>
